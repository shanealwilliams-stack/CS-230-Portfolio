# CS-230 Portfolio Reflection

Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design?

The Gaming Room was the client for this project. Their goal was to take their game, *Draw It or Lose It*, beyond Android and create a web-based application that could run on different operating systems. They wanted the game to support multiple users at the same time while being secure, dependable, and able to grow as the number of players increased.

What did you do particularly well in developing this documentation?

I think one of my strengths was clearly explaining my ideas and backing up my recommendations with logical reasons. Instead of only naming different technologies, I explained why they would be a good fit for the client's needs. I also made sure the document was organized and included every section required for the project.

What about the process of working through a design document did you find helpful when developing the code?

Completing the design document first helped me understand the project before writing any code. It gave me a chance to think through how everything would fit together and helped me organize my ideas. Having that plan made the coding process much smoother because I already knew how I wanted the application to be structured.

If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?

Looking back, I would spend more time improving the recommendations section. After getting feedback, I realized that explaining why I made certain recommendations was just as important as the recommendations themselves. Adding more detail made the document stronger and showed how each decision would benefit the client.

How did you interpret the user's needs and implement them into your software design? Why is it so important to consider the user's needs when designing?

I started by focusing on what the client expected the application to do and used those requirements to guide my design choices. Since they wanted software that could support different platforms and a large number of players, I selected solutions that emphasized performance, security, and scalability. Keeping the user's needs in mind is important because successful software should solve the client's problems while providing a positive experience for the people using it.

How did you approach designing software? What techniques or strategies would you use in the future to analyze and design a similar software application?

My design process began with reviewing the project requirements and breaking the project into smaller, more manageable tasks. I compared different operating systems, software architectures, storage solutions, and security options before deciding which ones best matched the client's goals. In future projects, I plan to continue using this approach because taking time to plan and evaluate different options before writing code leads to better software and fewer problems during development.
